核心洞察

团队有 PM、站会、review、retro 提供治理；
一个人 + AI 做开发时，这些全部消失。
Comet-Panel = 一个人的 PM + 团队记忆 + 知识管理，
用结构化数据和图谱关系替代团队协作带来的隐性治理。

架构哲学

Single Go Binary → Local-First → Filesystem as Truth → Incremental by Default → AI as Interface

vs 现有工具

vs GitHub/GitLab
那些是云端+团队+PR 流程；这是本地+个人+Change 生命周期+知识图谱
vs Obsidian
Obsidian 擅长记笔记；comet-panel 从变更工作流出发，再把知识纳入图谱
vs Linear/Jira
团队排期工具太重；收敛成面板+报告+图谱，无组织级噪音
vs CLI openspec list
CLI 只告诉"有什么"；Panel 还告诉"卡在哪、缺什么、谁关联谁"
